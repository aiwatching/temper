Here's a summary of the two bugs and the fix:

### Bug 1: Uncaught exceptions leave entities stuck (primary cause)
`processStage()` line 167 had no try-catch around `serviceFunction.apply(dto)`. If `PolicyService.evaluate()` threw any RuntimeException (NPE, DB connection error, etc.), the exception bypassed all retry/failure handling, and the entity stayed at LABELED with `processingCode=0` permanently. Recovery would pick it up every 80s but hit the same exception repeatedly.

**Fix:** Wrapped the service call in try-catch. Exceptions are now retried up to 3 times with backoff, then recorded as a failure with `code=-1` so the entity exits the state machine cleanly.

### Bug 2: Stage advanced unconditionally, even on failure
Lines 208-210 always set `processingStage=nextStage` regardless of success/failure. A LABELED entity that failed during policy evaluation would be written to DB as stage=EVALUATED with a non-zero code — masking where the actual failure occurred.

**Fix:** Stage is now only advanced when `serviceSuccess=true`. On failure, the entity stays at its current stage (LABELED) with the error code, making it clear where processing actually failed.
