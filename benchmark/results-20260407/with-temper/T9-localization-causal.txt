## Chain of Events: HA Failover Causing Random Logouts

Here's the full causal chain:

### 1. Failover triggers
When the primary HA node goes down, traffic shifts to the secondary node.

### 2. Connection pool exhaustion
The DB pool size was set to **50** (`AppConfig.java:9`). The constraint documented in the code is:

> `pool_size = max_connections / ha_node_count`

With 2 HA nodes, each node gets 25 connections. During failover, the secondary node suddenly receives **all** traffic — its pool of 50 connections gets exhausted under the combined load.

### 3. Session validation fails
`AuthService.validateToken()` needs a DB connection to verify sessions. With the pool exhausted, these queries time out, and token validation fails — returning users as unauthenticated.

### 4. No session pre-warming
`HAManager.failover()` has a comment acknowledging this: *"HA failover triggers session cache invalidation which causes all users to be logged out"*. The `preWarmSessions()` method exists but is unimplemented, so session state isn't copied to the secondary before switchover.

### Result
Every active user gets logged out simultaneously during any failover event.

---

## The Fix

**`AppConfig.java:9`** — DB pool size increased from `50` to `200`:

```java
// Before
return 50;

// After  
return 200;
```

With 200 connections and 2 HA nodes, each node gets 100 connections — enough headroom to absorb failover traffic without exhausting the pool. Session validation queries succeed, and users stay logged in.
