# 给 Claude Code 加了"记忆"后，省了 19% 的钱，还不会乱改代码了

姐妹们（不是）工程师们，今天测了一个让我很兴奋的东西 🔥

## 背景

用 Claude Code 写代码的人都知道一个痛点：**每次开新 session 它都是个新人**。不知道你们项目以前踩过什么坑，不知道哪些地方不能动，每次都要重新搜一遍代码。

我写了个工具叫 **Temper**，给 Claude Code 加上持久化记忆。把团队的经验、约束、设计决策存起来，下次 Claude 直接知道。

## 怎么测的

同一个任务，跑两遍：
- 一遍纯 Claude Code（没有任何记忆）
- 一遍带 Temper（有模块知识 + 约束 + 历史经验）

关键设计：**所有约束都不写在代码注释里**，只存在 Temper 的记忆中。模拟真实场景——很多规则只有老员工知道。

## 结果（10 个测试）

**总费用省了 19%，速度快了 18%**

但最炸裂的是这两个测试：

### 测试 1："给 DAO 加个缓存"

这个缓存以前在生产环境炸过——多节点部署时数据脏读。但这个教训没写在代码注释里。

- ❌ 无记忆：Claude 花了 6 轮一直在加缓存，最后超时了。**完全不知道有约束**
- ✅ 有记忆：3 轮直接拒绝，说"有明确约束不能在 DAO 加缓存，建议加在 Service 层"

**省了 30% 费用，快了 2 倍，还避免了生产事故。**

### 测试 2："加一个邮件通知功能"

- ❌ 无记忆：6 轮，59 秒，搜了很多文件才找到现有 pattern，花了 $0.19
- ✅ 有记忆：4 轮，17 秒，直接知道用 TemplateEngine，花了 $0.10

**快了 3.5 倍，省了 47% 的钱。**

## 总结

| | 无记忆 | 有记忆 |
|--|--------|--------|
| 费用 | $1.12 | $0.91 (-19%) |
| 耗时 | 241s | 197s (-18%) |
| 约束感知 | 72% | 86% (+14%) |

目前只在 25 个文件的小项目上测的。我的生产项目有 2000+ 文件，预计差异会更大（搜索噪音指数级增长）。

Temper 是 Rust 写的 MCP server，tree-sitter 做 AST 分析，SQLite 存知识。开源的。

有用 Claude Code 的可以试试，尤其是大型 legacy 项目。老项目的"暗知识"太多了，不记下来 AI 根本不知道。
